# Docs Site

`docs-site/` is the GOWDK documentation website. Authored docs live in the repo
root `docs/` tree; generated site pages are derived from those Markdown files.

## Edit Flow

1. Edit Markdown under `../docs` or the site source under `src/`.
2. Regenerate docs pages when Markdown structure changes.
3. Build with the in-tree `gowdk` CLI.
4. Do not commit `src/pages/docs/`, `src/components/docs-sidebar.cmp.gwdk`, or
   `dist/site` output unless a maintainer explicitly asks for generated output.

## Prerequisites

- Go `1.26.4` or the repository toolchain version.
- A local GOWDK CLI built from the repository:

```sh
(cd .. && go build -o docs-site/tools/gowdk ./cmd/gowdk)
```

- The Tailwind v4 standalone CLI at `tools/tailwindcss`. The build does not
  download Tailwind automatically.

## Common Commands

Run from `docs-site/`:

```sh
go run ./cmd/syncdocs
./tools/gowdk check
./tools/gowdk build
scripts/build-production.sh
scripts/smoke-production.sh
```

Run from the repository root for documentation-only changes:

```sh
scripts/check-docs-links.sh
scripts/check-docs-style.sh
scripts/check-removed-syntax.sh
scripts/check-doc-versions.sh
```

## Source Ownership

| Content | Edit here |
| --- | --- |
| Documentation prose | `../docs/**/*.md` |
| Generated documentation pages | generated from Markdown by `cmd/syncdocs` |
| Site layout and components | `src/components/`, `src/layouts/`, `src/pages/` |
| Global styling | `src/styles/app.css` |
| Production output | `dist/site/`, generated by build scripts |

`cmd/syncdocs` owns the docs sidebar and documentation page generation. Update it
when adding a new documentation section or changing docs-site routing rules.
