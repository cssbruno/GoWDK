# v0.1 Release Checklist

This checklist tracks the evidence needed before a release can claim the v0.1
target. The current `0.1.x` CLI line is still experimental until every item
marked as required is checked.

## Status Rules

- `[x]`: current repository docs and verification evidence support the item.
- `[ ]`: not done, not verified, or intentionally still blocked.
- Required: must be checked before the release may claim the v0.1 target.
- Production-ready: may remain unchecked for an experimental v0.1 release, but
  the release must not claim production readiness while any item is unchecked.

## Required Product Scope

### Build Output And Prerender

- [x] `gowdk build --out` emits app-shell HTML for simple build-time pages.
- [x] `gowdk build --out` emits `gowdk-routes.json`.
- [x] `gowdk build --out` emits `gowdk-assets.json`.
- [x] `gowdk build --out` emits `gowdk-build-report.json`.
- [x] Literal dynamic `paths {}` subsets generate concrete SPA route output.
- [x] Literal `build {}` data works for supported scalar fields.
- [x] Imported no-argument Go build data functions work.
- [x] Same-package no-argument Go build data functions work.
- [x] Build data supports earlier-field references.
- [x] Build data supports string concatenation.
- [x] Build data supports numeric arithmetic.
- [x] Build data supports boolean logic and comparisons.
- [x] Generated handlers needed for the full v0.1 build-output target are
  implemented or explicitly deferred with documented diagnostics.
- [x] Arbitrary build-time statements beyond expression records are implemented
  or explicitly deferred with documented diagnostics.
- [x] Full component semantics required for v0.1 examples are implemented or
  explicitly scoped down in requirements and docs.

### Compiler Contract

- [x] Portable package-peer `.gwdk` files compile through the typed GOWDK AST.
- [x] GOWDK analyzer lowers source files into normalized compiler metadata.
- [x] `internal/gwdkir` exists as the versioned compiler IR.
- [x] Build output generation consumes `internal/gwdkir.Program`.
- [x] App generation route planning consumes `internal/gwdkir.Program`.
- [x] CLI route and endpoint reports convert through `internal/gwdkir.Program`.
- [x] Remaining downstream compiler passes that still use compatibility structs
  are listed as planned or migrated to `internal/gwdkir`.
- [x] Parser, route, view, component, client, package, and build diagnostics
  have useful spans for every v0.1-supported language surface.
- [x] P0/P1 language constraints are documented: no arbitrary JavaScript as the
  app contract, no external template semantics, and no generated JavaScript as
  trusted business logic.
- [x] P0/P1 language constraints above have explicit compiler diagnostics or
  tests for each supported escape path.

### Routes And Rendering

- [x] Default render mode is `spa`.
- [x] Supported render modes are documented: `spa`, `action`, `hybrid`, `ssr`.
- [x] Request-time page behavior fails without the SSR feature enabled.
- [x] Dynamic SPA routes require `paths {}` unless using request-time rendering.
- [x] `load {}` selects request-time SSR and requires the SSR feature.
- [x] Hybrid route metadata exists internally; source syntax selects SSR with
  `load {}` or `go ssr {}`.
- [x] SSR support that is not production-complete is marked partial.
- [x] Hybrid streaming, data refresh, and non-HTTP revalidation are marked
  planned or partial.
- [x] Route conflicts and malformed route declarations are hard diagnostics.

### Actions, APIs, And Fragments

- [x] SPA pages can declare typed actions without SSR.
- [x] Generated action handlers cover the supported same-package handler

- [x] Generated action handlers support `response.Response` results.
- [x] Generated action decoding rejects unexpected fields.
- [x] Generated validation covers direct literal `required` controls.
- [x] Generated validation covers direct literal `minlength` controls.
- [x] Generated validation covers direct literal `maxlength` controls.
- [x] Generated validation covers Go-regexp-compatible `pattern` controls.
- [x] CSRF token generation and validation work when `Build.CSRF.Enabled` is

- [x] Direct file inputs and multipart generated action forms are rejected with

- [x] API declarations bind only to supported exported Go handlers.
- [x] Fragment routes and partial action responses use no-store request-time

- [x] Generated request-time lanes recover panics before response headers are
