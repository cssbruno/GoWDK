# v0.2 Release Checklist

This checklist tracks the v0.2 Public Truth and Release Trust bucket from
`docs/engineering/release-plan.md`. v0.2 is an experimental 0.x release target,
not a production-readiness claim.

## Status Rules

- `[x]`: current repository docs or observed release metadata support the item.
- `[ ]`: not done, not verified, or requires a GitHub-side/manual action.
- Release metadata and project board work may require GitHub UI or API actions
  outside the repository.

## Release Metadata Audit

- [x] `v0.1.5` release metadata was checked with:

  ```sh
  gh release view v0.1.5 --repo cssbruno/GOWDK \
    --json tagName,name,isDraft,isPrerelease,createdAt,publishedAt,url
  ```

  ```json
  {

    "name": "GOWDK v0.1.5",
    "isDraft": false,
    "isPrerelease": false,
  ```

- [ ] Update `v0.1.5` on GitHub so it is marked as a pre-release, or edit the
  release body to put the experimental/not-production-ready warning first.
- [ ] Verify future 0.x releases are created as draft and pre-release before
  publication.

## Repo Truth Gates

- [x] `docs/engineering/release-plan.md` describes v0.2 as Public Truth and
  Release Trust work.
- [x] `docs/engineering/release.md` links to the open-ended 0.x hardening
  checklist.
- [x] `.github/release-note-template.md` exists and requires experimental,
  not-production-ready, known gaps, checksum, and attestation sections.
- [x] `README.md` includes an experimental 0.x warning and support matrix.

## Release Notes Gates

- [x] Draft `docs/engineering/release-notes-v0.2.md` from

```sh

go build ./cmd/gowdk
node --check editors/vscode/extension.js

```
